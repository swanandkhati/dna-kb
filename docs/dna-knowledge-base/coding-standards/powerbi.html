Development Standards - PowerBI

This page elaborates the standards and guidelines for snowflake developers that they should be following when developing pages using PowerBI for DNA using snowflake.

All PowerBI development teams in DNA follow these standards to develop PowerBI pages and dashboards. The general convention is to not deviate from the set guidelines.

Should a need arise to deviate, kindly connect with the UI/UX team to discuss.

Typography​

Fonts​

Font to be usedSegoe UI
Font StylesRegular, Bold, Semi-Bold
Font DecorationsNone
Font Colors- For white/grey backgrounds: #1B2124
- For blue backgrounds: #FFFFFF
- For Page Titles: #009FDB
Font Sizing*Varies - Please refer to the individual images given below on this page.

Screen Size​

  • All PowerBI pages will be developed keeping a desktop view in mind as the primary mode of viewing and using the dashboards.
  • At the same time, the pages should also be developed by the front end team using responsive design. Please ensure your elements align with this philosophy.
  • Default desktop size would have a resolution of 1920 x 1080, and would adjust as per screen size from thereon.
  • All dashboards will be developed for a 100% zoom setting by default.
